# Proposal

## Why

当前桌面端已经具备 Skill 安装、平台分发、Rules 管理、MCP 配置等核心能力，但这些能力主要通过分散的按钮与表单暴露。对于高频用户，很多操作更适合通过一句自然语言指令交给内置 Agent Assistant 执行，例如“从这个链接安装 skill，并分发到 Claude / Codex / OpenCode”。这一需求需要先被明确记录为后续产品边界。

同时，Git 仓库导入弹窗在扫描出大量 Skill 候选时，可滚动结果区被仓库说明和提示块挤压得过小，用户难以快速判断还有哪些候选项，也不容易进行多选导入。

另外，SSH GitHub 仓库在扫描阶段仍错误地走了 GitHub API 元数据读取链路，导致用户明明输入了 `git@github.com:owner/repo.git`，却仍然触发匿名 API rate limit。这个行为与“SSH 应走本地 git/本地密钥”的预期不符，需要一并修正。

Git 仓库导入弹窗还有一个状态残留问题：用户改了仓库地址后，旧扫描结果仍然停留在界面上，缺少明确的“重新扫描”动作；如果用户返回再重新打开该模式，也会继续看到上一次的扫描态。与此同时，HTTP GitHub 限流报错缺少足够明确的“改用 SSH 可绕开匿名 API 限流”提示。

另外，Skill 详情页当前把内部 `currentVersion=0` 直接展示成了 `v0`。这虽然符合底层“尚未生成快照”的计数语义，但对用户来说没有意义；产品展示应从 `v1` 起显，不应出现 `v0`。

Skill 文件预览中的图片资源还缺少接近原生图片浏览器的缩放体验：用户在预览区滚动鼠标滚轮时应能直接放大/缩小，放大后应能通过抓手拖拽平移图片，缩放控件应贴近图片浏览区域而不是占用顶部文件工具栏，并且不能跟随图片内容一起滚动。“资源预览 / Preview” 这类状态文案也不应干扰用户查看图片。图片本体也不应被额外加阴影、圆角或边框感，否则带大面积白边的图片会看起来像被套了一层空白卡片。右下角百分比/全屏按钮应打开覆盖整个应用的大图预览，而不是被误解成“重置到 100%”。

Skill 详情页顶部标题是用户经常需要粘贴到其他工具中的名称；当前用户必须手动选中文本复制，操作成本偏高。标题本身应支持点击复制，并保留标题的视觉形态。

## Scope

- In scope:
  - 在 active change 文档中记录桌面端 `Agent Assistant` 的需求边界、能力来源和 MVP 方向。
  - 优化桌面端 “Install from Git Repository” 结果态布局，让候选列表获得更大的主滚动区域。
  - 为该弹窗的人机工学优化补充回归测试。
  - 修复 SSH GitHub 仓库扫描错误依赖 GitHub API 的问题，让扫描阶段改为本地 SSH clone 后解析。
  - 修复 Git 仓库导入弹窗的结果残留问题，并补上地址变更后的重新扫描入口。
  - 在 HTTP GitHub 限流报错中明确提示用户稍后重试或改用 SSH。
  - 调整 Skill 详情页当前版本展示，让首版统一从 `v1` 起显。
  - 优化 Skill 文件图片预览，把缩放控件移到图片区域右下角，支持鼠标滚轮缩放、抓手拖拽平移和全屏预览，并移除顶部“资源预览 / Preview”文案与图片本体阴影。
  - 支持在 Skill 详情页点击顶部标题直接复制 Skill 名称。
- Out of scope:
  - 本轮不实现 Agent Assistant 的执行引擎、气泡入口或聊天 UI。
  - 不改变 Git 仓库安装后的 source identity 规则，也不新增 GitHub OAuth / PAT 依赖。

## Risks

- 调整弹窗布局时，需要保持 footer 固定、按钮位置稳定，避免滚动区与 footer 抢高度。
- 压缩说明区时，不能让用户失去对支持 URL 类型或 README fallback 行为的理解。
- SSH clone 扫描需要维持与原远程扫描相同的 source metadata 结构，否则可能导致后续刷新、安装或分发行为出现 source mismatch。
- Git 导入模式在“返回 / 重新进入 / 修改地址”几个分支上的状态重置如果不一致，容易再次出现旧结果误导新导入目标的问题。
- 图片预览缩放如果只用 CSS transform，会导致放大后的可滚动尺寸不随图片变化，用户放大后仍然无法滚到图片边缘。
- 如果把缩放控件放在同一个滚动容器里，拖拽平移图片时控件会跟着内容一起移动，违背浮层控制的预期。

## Rollback Thinking

- Agent Assistant 需求文档仅为规划记录，回滚时直接移除 change folder 内容即可。
- Git 导入弹窗优化仅涉及 renderer 布局与测试；回滚可恢复组件 className 和测试断言。
- 图片预览优化仅涉及 `SkillFileEditor` renderer 组件与样式；回滚可恢复顶部缩放控件和原图片渲染规则。
